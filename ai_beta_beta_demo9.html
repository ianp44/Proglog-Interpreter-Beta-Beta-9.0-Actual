This is a beta beta demo (not fully functional) of a Javascript Prolog AI (Artificial Intelligence) language Interpreter.
Prolog is probably a bit out of date now, but can support a lot of Artificial Intelligence functionality.
An Interpreter is a (in this case Javascript) program that runs the Prolog program (from the Prolog program text typed in by the Prolog Programmer.)


This demo can create and display a rule list.
Also compiles and displays rule varable index list.
Also can test the match functor functor routines for entries
on the compute stack.
It should handle setting the variables correctly in the calling rule stack entry
but this is not tested yet - only have checked non-variable functor examples,
OK so far. (Have the flu 7/1/17).

This version can also load (but not modify) text files from the server.

About the Test Buttons
The test and display functor routines button creates and displays some functor trees, including with proper Prolog Variables.

The create rule base and display button creates some Prolog rules and puts them into the rule base, then displays the entire rule base.

The dump label table button displays all the internal program symbols from the label table.

To see the program output after you have clicked the buttons, scroll down the page.

The set rule button (a test button) selects the rule from the displayed rule list (only after you have
clicked the create rule base button) according to the top rule list index input value
and the rule list index input value.

The Try Match Button tests the compute stack entry functor match routines.
See blurb about it lower down at the button.


The point is that the rule list (that is all the rules, including facts with no tails) are organised as a list of lists (top rule list of rule lists) with the key concept being that all the rules in a rule list have
the same head functor identifier.
The selected rule is given (set) a entry on the compute stack, ready to be used by the matching routines to do the actual rule matching that makes Prolog work.

You can set as many rules as you like (to a preset limit) and then display the entire compute stack
by clicking on the display compute stack button.

The display compute stack button (a test button) displays the entire compute stack.
This includes the instantiated values of any variables, which have an link data entry
on the compute stack if they exist in the rule.

File Load Test Buttons
These can load prolog text files from the server, but not modify them.
At moment files are only test files.
See buttons for what they do.
